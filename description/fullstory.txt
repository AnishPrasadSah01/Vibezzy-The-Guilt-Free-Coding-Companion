

# **APP NAME: Vibezzy - The Guilt-Free Coding Companion**

## **APP STORY**

CodeBreak is a therapeutic coding environment designed for beginner developers who struggle with guilt around taking breaks and maintaining focus. Unlike traditional code editors that emphasize productivity at all costs, CodeBreak enforces healthy coding habits through structured focus-and-break sessions, validating rest as essential to the creative process.

The app combines a minimalist code editor with a visual progress system that gamifies the coding journey. During focus periods, users code in a distraction-free environment while motivational GIFs subtly encourage them. When break time arrives, the app doesn't just suggest a breakâ€”it *mandates* it, offering two paths: physical wellness activities or meme-based mental refreshment.

By forcing users into timed sessions and celebrating their completion, CodeBreak reframes coding as a sustainable marathon rather than an exhausting sprint. It's not about being the perfect IDEâ€”it's about creating a space where beginners can code, breathe, and actually enjoy the learning process.

## **APP DESCRIPTION**

**Purpose**: A beginner-friendly coding environment that enforces focus/break cycles with motivational content and guilt-free rest validation.

**Core Philosophy**: 
- Coding should be fun and sustainable
- Breaks are productive, not lazy
- Motivation through memes and validation, not pressure

**Key Features**:
- Customizable focus/break timer sessions (Pomodoro-style)
- Clean, distraction-free text editor
- Visual progress tracker with animated motivational overlays
- Mandatory break activities (physical tasks or meme browsing)
- Validating, encouraging messaging throughout

- On click: Save all values, navigate to Scene 3

### **SCENE 3: Main Coding Environment**

**Layout Structure**: 
Split-screen with 4:1 ratio (horizontal division)

#### **SECTION 1 (Left Side - 80% width): Code Editor**
- Plain text editor (textarea-style)
- No syntax highlighting, no compilation, no advanced features
- Just a clean space to write code
- Text persists during break sessions (saved in idle state)

#### **SECTION 2 (Right Side - 20% width): Vertical Progress Tracker**

**Visual Design**:
- Vertical column displaying focus/break blocks stacked top-to-bottom
- Number of blocks = Number of sessions Ã— 2 (focus + break per session)
- Blocks arranged as: Focus 1 â†’ Break 1 â†’ Focus 2 â†’ Break 2 â†’ Focus 3 â†’ Break 3

**Block Behavior**:

**FOCUS BLOCKS**:
- **Animation**: Background color fills from top to bottom over the focus duration
- **Overlay**: Motivational GIF plays on top of the filling background
  - GIFs stored locally in `/assets/motivation/` folder
  - Examples: thumbs-up animations, "you can do it" text GIFs, anime encouragement loops, coding-themed animations
  - GIFs should cycle/change dynamically during the focus period (like a slideshow)
- **Completion**: When focus timer expires, block disappears (deleted/removed from DOM)

**BREAK BLOCKS**:
- Become active only after their corresponding focus block completes
- Trigger the Break Window (see below)
- Background fills from top to bottom during break duration
- Completion: When break timer expires, block disappears

---

### **BREAK WINDOW (Interrupts Scene 3)**

When a focus block completes, the code editor enters **idle state** (background, not editable) and a new modal/overlay window appears.

**Break Window Layout**:
Two options presented as buttons/cards:

#### **OPTION 1: Physical Activity**
**Visual**:
- Activity suggestion displayed as centered quote/text
- Examples stored in a predefined list/JSON:
  - "Go to the kitchen and drink a glass of water"
  - "Step outside and get 5 minutes of sunlight"
  - "Go touch some grass (literally!)"
  - "Do 10 jumping jacks"
  - "Stretch your arms above your head for 30 seconds"

**Interaction**:
- "Done" button in bottom-right corner
- **Logic on Click**:
  ```
  IF remaining_break_time > 2 minutes:
      Display another physical activity
  ELSE IF remaining_break_time < 1 minute:
      Close break window
      Return to code editor (Section 1 active)
      Start next focus block timer
  ```

#### **OPTION 2: Meme**
**Visual**:
- Random meme image displayed center-screen
- Memes stored locally in `/assets/memes/` folder
- Pull from internet meme culture (funny, relatable coding memes preferred)

**Interaction**:
- "Next" button in bottom-right corner
- **Logic on Click**:
  ```
  IF remaining_break_time > 2 minutes:
      Display another random meme
  ELSE IF remaining_break_time < 1 minute:
      Close break window
      Return to code editor (Section 1 active)
      Start next focus block timer
  ```

**Timer Logic**:
- Break timer runs in background regardless of user's choice
- Both options share the same termination logic based on remaining time
